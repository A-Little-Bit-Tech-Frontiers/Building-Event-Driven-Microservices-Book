# 기본 프로듀서/컨슈머 마이크로서비스

기본 프로듀서/컨슈머(BPC: Basic Producer and Consumer)는 하나 이상의 이벤트 스트림에서 이벤트를 받아 필요한 비즈니스 로직을 적용해 가공하고 필요시 출력 이벤트 스트림에 이벤트를 내보내는 마이크로서비스

### BPC의 유스 케이스
- 기존 레거시 시스템과 통합 (e.g. 사이드카 패턴)
- 이벤트 순서와 무관한 상태 저장 비즈니스 로직 (e.g. 게이팅 패턴)
- 데이터 레이어가 너무 많은 일을 하는 경우
- 독립적으로 처리 및 데이터 레이어 확장

BPC 패턴은 단순하지만 강력하며 상태 저장/상태 비저장 이벤트에 수반한 수많은 마이크로서비스 패턴의 근간이다.  
매우 유연하다. 대부분의 비즈니스 처리 작업이 데이터 스토리지 레이어에서 처리되는 구현체와 궁합이 잘 맞다.  
이벤트 스트림과 레거시 시스템 간의 인터페이스 레이어로 사용할 수 있다.  

그러나 이 패턴은 워낙 기본적인 속성만 갖고 있어서 간단한 형태의 상태 구체화, 이벤트 스케줄링, 타임스탬프 기반의 의사결정 등의 메커니즘이 필요한 경우에는 관련 라이브러리에 투자해야 한다.
